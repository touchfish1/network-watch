## 项目概览

- 项目名称：`Network Watch`
- 产品定位：跨 `macOS / Windows / Linux` 的桌面悬浮监控工具
- 当前阶段：`v0.3.6，已进入持续打磨与自动发布阶段`
- 当前技术路线：`Tauri v2 + React + TypeScript + Rust(sysinfo)`

## 需求讨论记录

### 2026-04-13 初始技术选型讨论

讨论主题：桌面端 `macOS + Windows + Linux` 三端跨平台技术选型。

结论：

- 优先采用 `Tauri` 方案
- 目标产品不是重型任务管理器，而是轻量常驻桌面监控挂件
- 技术实现上采用 `Tauri v2 + React + TypeScript + Rust`

### 2026-04-13 Tauri 方向需求收敛

用户确认：

- 使用 `Tauri`
- 要做三端悬浮窗
- 用于监控 `CPU`、`内存`、`网络`
- 先做 web 调研，再讨论需求

经过调研与讨论后，需求被收敛为以下 MVP 规格：

- 监控范围：整机总览，不做按进程监控
- 窗口形态：角落小组件
- 首版能力：托盘 + 开机启动
- 前端栈：`React + TypeScript`
- 信息密度：两层视图
- 刷新频率：`1 秒`
- 交互方式：默认可交互，不默认鼠标穿透
- 展开态内容：数字 + 迷你趋势图
- 历史数据：仅内存短时趋势，不做落盘持久化

### 2026-04-13 收起态形态调整

新增讨论结论：

- 当前默认弹窗过大
- 默认形态改为固定在桌面边缘的紧凑状态条
- 状态条只显示一行关键指标
- 点击状态条后再展开为详细监控面板

### 2026-04-13 TrafficMonitor 风格收敛

新增讨论结论：

- 期望体验更接近 `TrafficMonitor`
- 默认应贴近任务栏使用
- 收起态的状态显示要完整，不应出现截断
- 拖动区域和展开按钮需要分离
- 真正“嵌入 Windows 任务栏内部”不作为当前跨平台版本的实现目标

### 2026-04-13 设置、升级与主题扩展

新增讨论结论：

- 需要在展开态增加设置区域，而不是只保留监控详情
- 需要支持基于 GitHub Release 的在线升级
- 需要支持主题切换，并优先提供中文语境更明显的风格主题
- 发布流程需要自动打包 `Windows / macOS / Linux`

### 2026-04-13 悬浮条与系统栏避让修正

新增讨论结论：

- 收起态必须尽量像系统状态条，不应出现双层背景壳
- 展开态不能被任务栏或顶部状态栏遮住
- 展开方向判断和吸边逻辑应统一基于显示器 `workArea`
- 收起态切换到其他应用后仍要保持常驻，不应被压到系统栏后面

## 已确认的产品规格

### 窗口与交互

- 窗口为无边框、置顶、透明背景的小组件
- 默认作为右上角悬浮窗使用
- 可拖动
- 支持记忆位置
- 默认可交互
- 关闭窗口时隐藏到托盘，而不是退出应用

### 监控指标

- CPU：整机使用率
- Memory：已用内存 / 总内存
- Network Download：整机下行实时速率
- Network Upload：整机上行实时速率

### 展示层级

- 收起态：展示一行状态条和核心指标摘要
- 展开态：展示更详细数值和迷你趋势图

### 数据策略

- Rust 侧负责采样
- 前端通过 Tauri 事件接收系统快照
- 采样频率固定为每秒一次
- 趋势数据只保存在内存中
- 历史长度保留最近 `300` 个采样点，约等于 `5 分钟`

## 已知限制

- 当前实现为“贴近任务栏的悬浮状态条”，不是真正嵌入任务栏内部
- 当前网络监控为整机总览，不区分网卡
- 当前不支持按进程监控
- 当前历史趋势只保存在内存中，不会落盘
- 当前未实现鼠标穿透
- Linux 不同桌面环境下的窗口表现可能存在差异
- Windows 下“始终压过系统栏”的行为仍依赖系统窗口层级，个别场景可能仍需继续微调
